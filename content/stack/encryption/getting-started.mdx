# Getting started

This guide takes you from zero to encrypting data in under 5 minutes.

## Prerequisites

- [Node.js](https://nodejs.org/) >= 18
- CipherStash initialized — complete the [Getting started](/stack/guides/getting-started) guide first

## Quick start

<Steps>
  <Step>
    ### Initialize your project

    Run the init wizard:

    ```bash
    npx @cipherstash/stack init
    ```

    The wizard will:
    - Authenticate you via browser-based device login (if not already authenticated)
    - Ask which database connection method you're using (Drizzle ORM, Supabase JS, Prisma, or Raw SQL)
    - Let you build an encryption schema interactively or use a placeholder
    - Generate your encryption client file (default: `src/encryption/index.ts`)
    - Install `@cipherstash/stack` and `@cipherstash/stack-forge` as dependencies
  </Step>

  <Step>
    ### Set up your database

    Run the interactive setup to install EQL in your PostgreSQL database:

    ```bash
    npx stash-forge setup
    ```

    Or use `stash-forge install` directly for non-interactive setup with flags like `--drizzle` or `--supabase`. See [Install and upgrade EQL](/stack/encryption/forge/install) for all options.
  </Step>

  <Step>
    ### Encrypt your first value

    The init wizard generated your schema and client. Now use them to encrypt data:

    ```typescript filename="src/index.ts"
    import { users } from "./encryption/schema"
    import { client } from "./encryption"

    const encryptResult = await client.encrypt("secret@squirrel.example", {
      column: users.email,
      table: users,
    })

    if (encryptResult.failure) {
      console.error(encryptResult.failure.type, encryptResult.failure.message)
    } else {
      console.log("Encrypted:", encryptResult.data)
    }
    ```

    Run it:

    ```bash
    npx tsx src/index.ts
    ```

    The result is a JSON payload (called a CipherCell) ready for database storage:

    ```typescript
    // Success
    { data: { c: '\\x61202020202020472aaf602219d48c4a...' } }

    // Failure
    { failure: { type: 'EncryptionError', message: '...' } }
    ```
  </Step>

  <Step>
    ### Decrypt data

    ```typescript filename="src/index.ts"
    const decryptResult = await client.decrypt(encryptResult.data)

    if (decryptResult.failure) {
      console.error(decryptResult.failure.message)
    } else {
      console.log("Plaintext:", decryptResult.data) // "secret@squirrel.example"
    }
    ```
  </Step>
</Steps>

<Callout type="info">
  For encrypting and decrypting multiple values or entire objects, see [Encrypt and decrypt](/stack/encryption/encrypt-decrypt) for bulk operations like `bulkEncryptModels` and `bulkDecryptModels`.
</Callout>

## Store in a database

Store encrypted data in any database that supports JSONB:

<Callout type="info">
  For searchable encryption in PostgreSQL, use the `eql_v2_encrypted` column type (installed by EQL in Step 3). See [Searchable encryption](/stack/encryption/searchable-encryption) for details.
</Callout>

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email jsonb NOT NULL
);
```

## Manual setup

If you prefer not to use the init wizard, you can set up manually:

<Steps>
  <Step>
    ### Install the SDK

    ```bash
    npm install @cipherstash/stack
    ```

    > If you're using Bun, version 1.3 or later is required.
  </Step>

  <Step>
    ### Define your schema

    Create `src/encryption/schema.ts`:

    ```typescript filename="src/encryption/schema.ts"
    import { encryptedTable, encryptedColumn } from "@cipherstash/stack/schema"

    export const users = encryptedTable("users", {
      email: encryptedColumn("email")
        .equality()
        .freeTextSearch()
        .orderAndRange(),
    })
    ```

    See [Schema definition](/stack/encryption/schema) for all index types.
  </Step>

  <Step>
    ### Initialize the client

    Create `src/encryption/index.ts`:

    ```typescript filename="src/encryption/index.ts"
    import { Encryption } from "@cipherstash/stack"
    import { users } from "./schema"

    export const client = await Encryption({ schemas: [users] })
    ```
  </Step>
</Steps>

Authentication is handled automatically by device-based auth. Make sure you have run `npx @cipherstash/stack init` first — see [Getting started](/stack/guides/getting-started).

<Callout type="warn">
  `@cipherstash/stack` includes a native FFI module. You must exclude it from bundlers like Webpack, esbuild, or Next.js. See [Bundling](/stack/encryption/bundling) for details.
</Callout>
