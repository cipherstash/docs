# CipherStash Documentation

CipherStash is the new standard for data security that feels invisible.
Encrypt, control, and audit access to sensitive data directly in your TypeScript applications.

```bash
npx @cipherstash/cli init
```

Run `npx @cipherstash/cli init` to set up device-based authentication — no environment variables needed for local development.
Each developer gets a unique identity tied to their machine. See the [Getting started](/stack/guides/getting-started) guide to begin.

## Choose your integration

CipherStash offers a few ways to leverage the data security stack:

| | TypeScript SDK | CipherStash Proxy |
|---|---|---|
| **How it works** | Encryption and Secrets SDK and CLI available as an NPM package | Postgres Encryption - Postgres wire protocol Proxy built on top of the Encryption SDK to transparently encrypt data in Postgres |
| **Best for** | Teams moving quickly and building TypeScript apps |  More complex app architectures or non TypeScript applications using Postgres |
| **Language** | TypeScript / JavaScript (Node.js, Deno, Bun) | Any language (connects via PostgreSQL wire protocol) |
| **Setup** | `npx @cipherstash/cli init` | Docker container or binary |

Most teams start with the **SDK** for the best developer experience.

## How it fits together

Every CipherStash product builds on [ZeroKMS](/stack/kms) and its core primitive, [Key Sets](/stack/kms/keysets):

- **[Encryption](/stack/encryption)** uses Key Sets for [tenant isolation](/stack/encryption/configuration#keysets). Each tenant gets its own cryptographic boundary.
- **[Secrets](/stack/secrets)** uses Key Sets for [environment isolation](/stack/secrets/concepts#environments). Production, staging, and development secrets are cryptographically separated.
- **[Proxy](/stack/proxy)** encrypts data transparently via PostgreSQL, backed by the same ZeroKMS key hierarchy.
